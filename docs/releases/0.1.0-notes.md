# v0.1.0 Release Notes

## Added
- `GET /claims/{claimId}` returns claim status by id (REQ-300), `GET /claims` returns the full seeded claim list (REQ-301), `GET /health` for liveness/readiness probes (REQ-302).
- Domain model + `IClaimsRepository` seam (ADR-002), backing the three endpoints above.
- Multi-stage, non-root Dockerfile (.NET SDK build → ASP.NET runtime), listening on port 8080 (REQ-303).
- Bicep templates provisioning Azure Container Registry (admin access disabled) and AKS (system-assigned managed identity, RBAC enabled, `AcrPull` grant) — no static registry credentials in infra or CI (REQ-304, ADR-003).
- Kubernetes `Deployment` with liveness/readiness probes on `GET /health`, CPU/memory `requests`/`limits`, in a dedicated `claims-api` namespace (REQ-305).
- Kubernetes `Service` of type `LoadBalancer` exposing the Deployment on port 80 → container port 8080 (REQ-306, ADR-004 — public exposure accepted for v1, see Security below).
- GitHub Actions workflow: `dotnet restore`/`build`/`test` plus a dependency vulnerability scan on every push/PR, independent of Azure access (REQ-307); OIDC-gated build/push/deploy stages that run only when Azure credentials are present, skipped (not failed) otherwise (REQ-308).

## Security
SECURITY review at `docs/security/0.1.0-review.md`, disposition at `docs/security/0.1.0-disposition.md`. 5 findings: 0 critical, 0 high, 4 medium, 1 low. SECURITY exit gate (`art-threat-driven-security`) satisfied without the `impl/sec-<finding-id>` back-edge.

- **Accepted:** SEC-001 (unauthenticated read access to synthetic claim data — ADR-004's deliberate v1 boundary, expires 2026-12-31), SEC-002 (no rate limiting — out of scope per spec's Won't Have, K8s resource limits bound blast radius; low, recorded for traceability).
- **Deferred:** SEC-003 (base image not digest-pinned, expires 2026-09-30), SEC-004 (vulnerability scan step doesn't fail the build, expires 2026-09-30), SEC-005 (AKS API server has no `aadProfile`/authorized IP ranges, expires 2026-09-30).

A second-reviewer confirmation of the provisional severities (especially the SEC-001/SEC-002 decomposition of ADR-004's original "high" exposure call) is recorded as outstanding in the verifier findings on PR #6 — see `docs/security/0.1.0-disposition.md` header.
