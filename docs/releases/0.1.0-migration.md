# Migration to v0.1.0

First product release of the Claims Status API. No prior product version exists
to migrate from (`0.0.1-demo` was a pipeline demonstration of an unrelated
greeting function, not this product). No migration steps.

## Fresh deployment

1. Provision infra: `az deployment group create` against `infra/bicep/main.bicep` (provisions ACR + AKS with managed-identity `AcrPull`; no static credentials required).
2. Configure the GitHub Actions OIDC secrets (`AZURE_CLIENT_ID`, `AZURE_TENANT_ID`, `AZURE_SUBSCRIPTION_ID`) on the repository so the build/push/deploy stages run (REQ-308). Without them, CI still runs build/test/scan but skips deploy.
3. Push to `main` (or open a PR) to trigger `.github/workflows/claims-api.yml`, which builds the image, pushes to ACR, and applies the Bicep + Kubernetes manifests.
4. Confirm `GET /health` returns `200 OK` against the LoadBalancer's external IP once the Service has provisioned one.
